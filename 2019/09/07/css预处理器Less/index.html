less

一：认识less

1：什么是less？
less是一种动态样式语言，属于css预处理，它扩展了css语言，增加了变量，混合（mixin），函数等特性，更易于维护和扩展。
2：less的运行环境？
less可以在客户端上运行，也可以借助Node.js在服务器端运行。

二：less的编译工具

koala 官网：http://www.koala-app.com

三：less中的注释

1：以//开头的注释，不会被编译到css文件中
2：以/**/包裹的注释会被编译到css文件中

四：less中的变量

使用@来声明一个变量：@w
less中引用混合文件：@import "triangle.less";
1.属性值为变量
原：background：pink；
@c：pink；
现（加变量）：background：@c；
2.属性名为变量
原：margin：0；
@m:margin；
现：@{m}：0;
3.选择器为变量
原：.warp{}
@selector：.wrap；
现：@{selector}{}

五：less中的嵌套规则

1.父子嵌套（基本嵌套）
.warp {
父元素代码
.inner {
.juzhong(100px,100px,pink);
}
}
2.&的使用
同级嵌套&：hover==.wrap：hover

六：less的混合

一般将混合单独写入一个文件中，混合越复杂，业务代码越简单。
什么是混合？
混合就是将一些列属性从一个规则及引到另一个规则集的方式。
个人理解（通俗说就是，将公共代码，或者将要加变量，匹配方法放在一起组成一个混合。）

混合的分类

1.普通混合
写法：.juzhong{}；
调用：.juzhong;
2.不带参数的混合
写法：.juzhong{};
问题：css文件中会出现.juzhong{}；的代码
解决：让其不在css中出现加()—.juzhong(){};
3:带参数的混合
用法：.juzhong(@h,@w,@c){}
调用：.juzhong(100px,100px,pink);
4.带默认值的混合（没有传值时，有默认样式）
用法：.juzhong(@h=10px,@w=10px,@c=#bfa){}
调用：.juzhong(@w:100px);
5.命名参数
调用：.juzhong(@w:100px);
调用时，想要其他属性默认，然后传指定参数值
6.匹配模式
在混合中定义多种匹配选项，每个匹配项都有自己的样式和规则。（例子中为三角形方向问题：定义了4个规则）（每一个匹配项都有一个匹配符），根据需要调用匹配项。
①：定义匹配模式
第一个：.triangle(L,@w,@c){}；
第二个：.triangle(R,@w,@c){}；
然后在业务代码中调用
.triangle(T,50px,blue);

七：less的继承

1：less的继承不能有变量，混合都是不带参数的。

2：为什么会有继承？
因为一般的混合，编译less后的css代码中任然会有一部分的重复代码，这时想让这些重复代码合在一起—此时混合无法做到。

例：wrap父元素下有两个inner子元素，两个子元素都是仔wrap中垂直水平居中的。此时两个inner的垂直水平居中就是重复代码。
所以继承为：

.wrap {
     position: relative;
     width: 300px;
     height: 300px;
     border: 1px solid;
     margin: 0 auto;
     .inner{
         &:extend(.juzhong all);

         &:nth-child(1) {
             width: 100px;
             height: 100px;   
             background: pink;
         }
         &:nth-child(2) {
            width: 50px;
            height: 50px;
            background: yellow;
        }
    }
}

&:extend(.juzhong all);继承所有juzhong
